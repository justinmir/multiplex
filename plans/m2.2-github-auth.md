# M2.2 — GitHub OAuth + token storage (electron-store ConfigStore)

**Phase:** 2 — Git Integration
**Depends on:** M2.1

## Goal
Build an Electron-based GitHub OAuth flow. User clicks "Connect GitHub" in app → system browser opens for auth → local server captures callback code → exchanges for token via GitHub API → stores securely in electron-store ConfigStore. Expose via IPC channels.

## Changes

### 1. Add dependency: `electron-store` to packages/main
- Used as `ConfigStore` — key-value store backed by JSON in userData directory

### 2. New file: `packages/main/src/git/GitHubAuth.ts`
- `startOAuth()` method: opens system browser (`shell.openExternal`) to GitHub OAuth authorize URL with device-specific client ID + callback port
- Spawns a local HTTP server on an ephemeral port to capture the authorization code
- Exchanges code for token via GitHub's device/token endpoint

### 3. New file: `packages/main/src/git/ConfigStore.ts`
- Thin wrapper around `electron-store` providing typed access:
  - `getGitHubToken(): string | null` / `setGitHubToken(token: string)`
  - `isGitHubConnected(): boolean`
  - Future extensibility for other credentials

### 4. Extended IpcContract (packages/core/src/ipc.ts):
- `"github:get-token": { req: void; res: string | null }` — returns stored token or null
- `"github:connect": { req: void; res: { success: boolean } }` — initiates OAuth flow
- `"github:get-status": { req: void; res: { connected: boolean } }` — check connection status

### 5. New IPC handlers in `packages/main/src/ipc/handlers/github-auth.ts`
- Wire ConfigStore + GitHubAuth into IPC channels

## Implementation Notes
- OAuth callback port (39876) is fixed to avoid dynamic port issues with GitHub's redirect URL whitelist
- CSRF protection via state parameter ensures callback came from our auth request
- 5-minute timeout prevents server hanging if user abandons the flow
- `shell.openExternal()` opens system browser — bypasses Electron's webContents security handlers (which block external URLs)
