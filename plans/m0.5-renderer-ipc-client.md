# M0.5: Renderer IPC client + useIpc hook + dev ping button

## Goal
Typed renderer wrapper over `ipc.invoke`/`subscribe`, proven with a round-trip via dev PingProbe widget.

## Changes
1. **New file: `packages/renderer/src/lib/ipc/client.ts`** — typed `call()` (request/response) and `on()` (event subscription) wrappers around preload's `ipc.invoke` / `ipc.subscribe`
2. **New file: `packages/renderer/src/lib/ipc/useIpc.ts`** — React hooks: `useInvoke` for async IPC calls (auto-execute or manual via deps), `useSubscribe` for event listening
3. **New file: `packages/renderer/src/lib/dev/PingProbe.tsx`** — temporary dev widget (fixed bottom-right) that pings `app:ping` and displays the response; to be removed in M0.7
4. **Edit: `packages/renderer/src/main.tsx`** — mount `<PingProbe />` alongside `<App />`
5. **Config: renderer vite.config.ts** — add `@app/preload` alias for type resolution
6. **Config: renderer package.json** — add `@app/preload` as devDependency

## Validation
- `npm run validate` passes (0 errors)
- `npm run screenshot` boots app with no console errors
