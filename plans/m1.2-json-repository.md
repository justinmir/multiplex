# M1.2 — JsonRepository (userData-backed), first-run seed

## Goal
Create a durable JSON store under Electron `userData`, seeded from mock data on first launch. Extends InMemoryRepository with persistence.

## Files created

### `packages/main/src/repo/paths.ts`
- Resolves userData path via `app.getPath("userData")` + `/multiplex/db.json`
- Exports `DB_PATH`, `ensureDir()`, and `dataDir()`

### `packages/main/src/repo/seed.ts`
- Standalone seed data (projects + standalone sessions) extracted from renderer's mockData
- No renderer dependency — main process only imports from core types
- Data is static; timestamps are computed at module load via `Date.now()`

### `packages/main/src/repo/JsonRepository.ts`
- Extends InMemoryRepository with load/save to JSON file
- Constructor: loads existing db.json or seeds + saves on first run
- Every mutating method overridden to call `saveToDisk()` after mutation
- Atomic writes via temp file + rename
- Corrupt JSON guard: backup bad file, re-seed

## Design decisions
- Serialization format: maps serialized as arrays (data is small scale)
- Optional path parameter on constructor for testing flexibility
- All imports use `.js` extensions (ESM in TypeScript)
