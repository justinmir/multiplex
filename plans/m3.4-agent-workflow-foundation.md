# M3.4 — Agent Workflow Foundation

## Goal
Wire the composer Send button to persist user messages via IPC, and build an agent execution stub that simulates agent responses with status transitions.

## Changes

### 1. Extend IpcContract (`packages/core/src/ipc.ts`)
Add three new channels:
- `sessions:add-message` — persist a message to a session's conversation
- `agents:start` — start simulated agent execution for a session
- `agents:stop` — stop agent execution, mark session completed

### 2. Create handler file (`packages/main/src/ipc/handlers/session-messages.ts`)
Three handlers using the same pattern as `session-writes.ts`:
- `sessions:add-message`: get session, append message, upsert, emit data:changed
- `agents:start`: set status to "running", simulate agent response after 1.5s delay with "Processing your request..." message, set status to "completed"
- `agents:stop`: set status to "completed", upsert

### 3. Wire handlers (`packages/main/src/modules/IpcModule.ts`)
Import and call `registerSessionMessageHandlers(repo)` in enable().

### 4. Extend DataSource interface (`packages/renderer/src/lib/data/types.ts`)
Add three methods matching the IPC channels.

### 5. Implement IpcDataSource (`packages/renderer/src/lib/data/IpcDataSource.ts`)
Simple IPC call wrappers for each new channel.

### 6. Update MockDataSource (`packages/renderer/src/lib/data/MockDataSource.ts`)
Minimal no-op stub implementations to satisfy TypeScript interface.

### 7. Extend DataProvider mutations (`packages/renderer/src/lib/data/DataProvider.tsx`)
- `addMessage`: optimistic update (append message to local session), call IPC, rollback on error
- `startAgent`: call IPC directly (agent stub handles all state transitions)
- `stopAgent`: optimistic status change to "completed", call IPC, rollback on error

### 8. Wire SessionDetail (`packages/renderer/src/app/components/SessionDetail.tsx`)
Add two new props: `onSendMessage(sessionId, message)` and `onStopAgent(sessionId)`
- Composer Send button (when session exists): create user message, call onSendMessage, then onSendMessage triggers agent via startAgent in parent
- Top bar Stop button: call onStopAgent

### 9. Wire TaskView (`packages/renderer/src/app/components/TaskView.tsx`)
Pass through new callbacks to SessionDetail.

### 10. Wire AppShell (`packages/renderer/src/shell/AppShell.tsx`)
Implement sendMessage and stopAgent handlers using DataProvider mutations, pass to TaskView.

## Testing
- `npm run validate` must pass (typecheck + lint + build)
- Manual: send a message in session view → agent responds with simulated reply after delay
- Manual: click Stop button → session status changes to completed
