# M2.3 — GitHub API Client (Octokit) for PRs/Checks/Reviews

## Summary
Add Octokit-based GitHub REST API client to fetch real PR data, check runs, and review verdicts from GitHub repositories. Exposes two new IPC channels (`prs:list` and `checks:get`) so the renderer can query live GitHub data.

## Changes
1. **packages/main/package.json** — Add `@octokit/rest` dependency
2. **packages/core/src/ipc.ts** — Extend IpcContract with `prs:list` and `checks:get` channels  
3. **packages/main/src/git/GitHubMapper.ts** — Pure mapper functions (GitHub API → domain types)
4. **packages/main/src/git/GitHubClient.ts** — Octokit wrapper class for PRs/checks/reviews
5. **packages/main/src/ipc/handlers/github.ts** — IPC handler registrations for new channels
6. **packages/main/src/modules/IpcModule.ts** — Wire `registerGitHubHandlers()` in enable()

## Design Decisions
- All API methods return empty arrays/defaults on failure (graceful degradation without token)
- Octokit pagination capped at 5 pages (500 PRs max) for performance
- Token cached on first getOctokit() call to avoid recreating client
- Mapper functions are pure and importable/testable independently
