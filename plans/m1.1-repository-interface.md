# M1.1 — Repository interface + InMemoryRepository

## What was built
- **`packages/core/src/repository.ts`** — `Repository` interface defining the full storage contract (projects, sessions, notes, references, activity, archive). Pure types only; no Node/Electron imports.
- **`packages/core/src/index.ts`** — barrel export updated to re-export `repository.js`.
- **`packages/main/src/repo/InMemoryRepository.ts`** — Map-backed in-memory implementation of `Repository`. Uses defensive cloning on all reads. Supports project-scoped and session-scoped references via composite keys.

## Design decisions
- All methods are async (return Promises) for interface consistency, even though the in-memory impl is synchronous internally.
- Notes use `${projectId}::${noteId}` as map keys; references use `proj:${projectId}::${refId}` or `sess:${sessionId}::${refId}` depending on scope.
- A simple numeric counter generates IDs (`prefix_${++counter}`) — no crypto needed for test/reference impl.
- Maps are `protected` so a future `JsonRepository` can extend and persist them.
