# M6.2: Realtime Fan-out and Live Triage Ordering

## Problem
HomeView's "Needs you" section requires full `loadData()` reload when session statuses change, causing latency in reordering by weight. All components consuming session data get stale state until the broad `"data:changed"` triggers a complete refresh cycle.

## Solution
1. **Granular emit** — After `deriveSessionStatus` changes a session's status, emit `session-status-changed` with `{ sessionId, status }` alongside the existing `"data:changed"`.
2. **Optimistic subscription** — DataProvider subscribes to `session-status-changed` and does an optimistic local state update (patch one session in array), triggering immediate React re-renders without full IPC reload.
3. **useLiveData hook** — Expose fine-grained event subscriptions for components needing per-session reactions.
