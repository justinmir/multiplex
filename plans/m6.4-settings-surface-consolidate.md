# M6.4 — Settings surface (consolidate)

**Phase:** 6 — Status model, triage, realtime, search, settings  
**Depends on:** No strict dependency, but builds on existing ConfigStore and SettingsDialog from Phase 2-4.

## Goal
A single consolidated Settings panel: active harness + options, model defaults, provider/API tokens, GitHub token, repo roots, intelligence toggles — all persisted and reachable from the UI gear icon.

## Why
Currently settings are scattered: `ConfigStore.ts` (electron-store) holds only `githubToken`, `SettingsDialog.tsx` shows a minimal "Connect GitHub" button with no other configuration options. Users need to configure harness selection, API keys, model defaults, and repo roots — but there's nowhere in the UI to do it.

## Current state
- **ConfigStore (main):** Uses `electron-store`, stores only `{ githubToken?: string }`
- **SettingsDialog (renderer):** Minimal dialog with GitHub connection status button only
- **No Settings IPC channels:** No `settings:get`/`settings:set` in IpcContract yet
- **Gear icon exists:** Sidebar footer has a gear/settings button that opens the existing dialog

## Files
- `~ packages/core/src/ipc.ts` — add settings IPC channels
- `+ packages/main/src/settings/AppSettings.ts` — consolidated settings store and getter/setter
- `~ packages/main/src/modules/IpcModule.ts` — register settings handlers
- `~ packages/renderer/src/shell/SettingsPanel.tsx` — full settings UI (replace or extend SettingsDialog)
- `~ packages/renderer/src/app/components/SettingsDialog.tsx` — update to use new SettingsPanel

## Steps

### 1. Define AppSettings shape and IPC channels

In core, add IPC contract entries:
```typescript
// packages/core/src/ipc.ts — append to IpcContract
"settings:get": { req: void; res: AppSettings };
"settings:set": { req: Partial<AppSettings>; res: AppSettings };
```

### 2. Create `packages/main/src/settings/AppSettings.ts`

Replace ConfigStore with a richer settings model, persisted via the same electron-store or as part of Repository:
```typescript
import Store from "electron-store";
import type { HarnessConfig } from "@app/core";

export interface AppSettingsData {
  // Harness configuration
  harness?: HarnessConfig;
  
  // Model defaults
  defaultModel?: string;
  
  // Provider tokens (never sent to renderer — only used by main)
  anthropicApiKey?: string;
  githubToken?: string;
  
  // Repo roots for workspace selection
  repoRoots: Array<{ name: string; root: string }>;
  
  // Intelligence toggles
  intelligenceEnabled: boolean;
  autoSynthesizeOnActivity: boolean;
  dailySynthesisHour: number;  // 0-23
  
  // UI preferences  
  theme?: "dark" | "light";
}

const defaultSettings: AppSettingsData = {
  repoRoots: [],
  intelligenceEnabled: false,
  autoSynthesizeOnActivity: true,
  dailySynthesisHour: 6,
};

export class AppSettings {
  private store: Store<AppSettingsData>;
  
  constructor() {
    this.store = new Store<AppSettingsData>({ name: "multiplex-settings" });
  }
  
  get(): AppSettingsData {
    return { ...defaultSettings, ...this.store.store };
  }
  
  set(partial: Partial<AppSettingsData>): AppSettingsData {
    this.store.set(partial);
    return this.get();
  }
}
```

### 3. Register IPC handlers in IpcModule

Wire `settings:get` and `settings:set` to the AppSettings instance:
```typescript
handle("settings:get", () => appSettings.get());
handle("settings:set", (req) => {
  const updated = appSettings.set(req);
  // Emit so other services can react (e.g., SessionRuntime picks up new harness config)
  emit("settings:changed", {});
  return updated;
});
```

### 4. Build SettingsPanel.tsx (renderer)

Replace the minimal dialog with a tabbed or sectioned settings panel:
```tsx
// Sections:
// - "Harness": dropdown for mock/opencode, test connection button
// - "Models": default model input  
// - "API Keys": masked inputs for Anthropic/GitHub tokens (never sent to renderer — only written via IPC)
// - "Repos": add/remove repo roots with validation
// - "Intelligence": toggles for auto-synthesize, daily synthesis hour

import { useDataProvider } from "../data/DataProvider";
import { call } from "../../lib/ipc/client";

export function SettingsPanel({ open, onOpenChange }: Props) {
  const [settings, setSettings] = useState<AppSettingsData>({ repoRoots: [], intelligenceEnabled: false, ... });
  const [saving, setSaving] = useState(false);
  
  // Load settings on mount
  useEffect(() => {
    if (open) call("settings:get", undefined).then(setSettings);
  }, [open]);
  
  const save = async (partial: Partial<AppSettingsData>) => {
    setSaving(true);
    try {
      await call("settings:set", partial);
      toast.success("Settings saved");
    } catch (e) {
      toast.error("Failed to save settings");
    }
    setSaving(false);
  };
  
  // Render grouped form sections...
}
```

Key UI considerations:
- Token fields are masked (`type="password"`) with a show/hide toggle
- "Test connection" buttons call health checks (harness.health(), GitHub auth probe)
- Repo roots have add/remove with `git rev-parse` validation via IPC
- Persisted on every change (auto-save) or explicit Save button

### 5. Wire SettingsDialog to use SettingsPanel

Update the existing SettingsDialog to render the new panel when opened, keeping the same trigger mechanism (gear icon in sidebar).

## Acceptance
- Opening settings shows all configuration sections: harness, models, API keys, repos, intelligence
- Changing a setting persists across app restart
- Token fields are masked; repo roots validate as real git repos before saving
- "Test connection" for GitHub shows success/failure feedback
