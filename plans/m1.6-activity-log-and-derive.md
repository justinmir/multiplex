# M1.6: Activity log writes + lastActivity recompute

## Goal
On note/ref/session writes, append `ActivityItem` entries to the project's activity log.
Create a derive module that computes `openPRs`, `activeSessions`, and `lastActivity` from real
session/PR timestamps — applied when returning a project via IPC read handlers.

## Changes

### 1. New: `packages/main/src/repo/derive.ts`
Pure computation module with no side effects:
- `deriveProjectFields(p: Project): Project` — recomputes derived fields on a project copy
- `computeLatestTimestamp(project: Project)` — scans session timestamps for the most recent activity
- `formatRelativeTime(ts: number)` — converts epoch ms to human-readable relative string ("12m ago")

### 2. Updated: `packages/main/src/ipc/handlers/writes.ts`
Each write handler now appends an `ActivityItem` after its mutation:
- **notes:upsert** → `"Note <title> updated"` activity item on the project's log
- **notes:delete** → `"Note deleted"` activity item
- **refs:upsert** → `"Reference added — <title>"` (only for project-scoped refs)
- **refs:delete** → `"Reference removed"` (only for project-scoped refs)
- **sessions:archive** → skipped for standalone sessions (no projectId available)

### 3. Updated: `packages/main/src/ipc/handlers/repo.ts`
Project read handlers now wrap results with `deriveProjectFields`:
- `projects:list` — each project in the list gets derived fields applied
- `projects:get` — returned project (if found) gets derived fields applied

## Design decisions
- derive.ts is synchronous and pure — no async, no I/O
- Activity IDs use timestamp + random suffix for uniqueness within same ms
- Session timestamps (`createdAtMs`) are the primary source for lastActivity computation
- Standalone session archive skips activity logging (Phase 2 concern)
- PR `updatedAt` is already a relative string in domain.ts — not usable for timestamp computation
