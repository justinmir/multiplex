# M6.3 — Global Search (⌘K) Palette with cmdk

## Goal
Wire a global search palette using cmdk so ⌘K opens a searchable overlay over real projects and sessions. Selecting a result navigates to that entity.

## Changes

### 1. packages/core/src/ipc.ts
Add `search:query` IPC channel to IpcContract for future-proofing (server-side handler not needed yet — filtering is client-side).

### 2. packages/renderer/src/lib/search/SearchPalette.tsx (new file)
Cmdk-based CommandDialog palette that searches DataProvider's loaded data:
- Combines standalone sessions + project-scoped sessions into one flat list
- Filters by query text against title and prompt/description fields
- Groups results by "Sessions" and "Projects"
- Handles selection to navigate to the entity (setSelectedProjectId / setSelectedSessionId)

### 3. packages/renderer/src/shell/AppShell.tsx
- Add `useState` for search dialog open state
- Add `useEffect` keyboard listener for ⌘K / Ctrl+K and Escape
- Render `<SearchPalette>` at the bottom of AppShell alongside other overlays
- Pass navigation callbacks so SearchPalette can route to selected entities

## Navigation Model
Since Session type has no projectId field, project-scoped sessions are flattened with a `projectId` tag from their parent Project. On selection:
- Session → setSelectedSessionId + open "session" view; if project-scoped, also select the project and open "project" view
- Project → setSelectedProjectId + open "project" view

## Testing
- `npm run validate` must pass (typecheck + lint + build)
- Manual: ⌘K opens palette, type to filter, click to navigate
