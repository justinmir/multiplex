# M4.2 — Settings Panel + GitHub Connect Flow

**Phase:** 4 — Polish & Usability
**Depends on:** Phase 3 complete (M3.1–M3.4)

## Goal
Make the GitHub OAuth flow reachable from the UI by building a settings panel and wiring the existing `github:connect` IPC channel. Currently the entire Phase 2 auth infrastructure is built but unreachable.

## Current State
- Settings gear icon in ProjectsSidebar.tsx (line ~261): no onClick handler
- "Connect GitHub for live PRs" banner in ProjectsSidebar.tsx (lines ~267-270): non-clickable span
- `github:connect` IPC channel exists with handler but renderer never calls it
- No settings dialog/panel component exists

## Changes

### 1. New SettingsDialog component (`packages/renderer/src/app/components/SettingsDialog.tsx`)
Build a shadcn Dialog-based panel with:
- GitHub connection status indicator (green dot if connected, gray if not)
- "Connect GitHub" button → calls `connectGitHub()` mutation from DataProvider
- Uses relative imports for shadcn components (`./ui/dialog`, etc.)

### 2. DataSource interface + implementations
Add method: `connectGitHub(): Promise<{ success: boolean }>` to:
- `types.ts` (interface)
- `IpcDataSource.ts` (calls `"github:connect"` IPC channel)
- `MockDataSource.ts` (returns `{ success: false }`)

### 3. DataProvider mutations
Expose `connectGitHub()` in DataMutationValue, refresh GitHub status on completion

### 4. Wire ProjectsSidebar.tsx
- Settings gear icon → open SettingsDialog via new `onOpenSettings` callback prop
- "Connect GitHub" banner → clickable button that opens settings dialog

### 5. Wire AppShell.tsx
- Add `settingsOpen` state + render `<SettingsDialog>` at the root level

## Validation
- `npm run validate` passes (0 errors)
- `npm run screenshot` boots without errors
- Settings gear icon is clickable and opens a settings dialog
