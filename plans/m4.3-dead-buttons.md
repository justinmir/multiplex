# M4.3: Wire dead buttons in SessionDetail — Stop, Merge PR, Open on GitHub

## Problem
SessionDetail has three UI elements with polished styling but no click handlers:
1. **Stop button** (line ~112) — `onStopAgent` prop passed through but does nothing in App.tsx
2. **"Open on GitHub"** link (PR card, line ~305) — `<button>` with icon only
3. **Merge PR** button (PR card, lines ~308-316) — has `canMerge` logic but no onClick

## Solution

### IPC layer (core + main process)
1. **IpcContract** (`packages/core/src/ipc.ts`) — add two new channels:
   - `prs:merge`: merge a PR via Octokit
   - `app:open-url`: open URL in system browser via Electron shell

2. **GitHubClient** (`packages/main/src/git/GitHubClient.ts`) — add `mergePR(owner, repo, prNumber)` method using `octokit.pulls.merge()`

3. **github.ts handlers** (`packages/main/src/ipc/handlers/github.ts`) — register handler for `prs:merge` channel

4. **app.ts handler** (new: `packages/main/src/ipc/handlers/app.ts`) — handle `app:open-url` using `shell.openExternal(url)` from Electron

5. **IpcModule** (`packages/main/src/modules/IpcModule.ts`) — register the new app-level handler

### Renderer data layer
6. **DataSource types** (`packages/renderer/src/lib/data/types.ts`) — add `mergePR()` and `openUrl()` methods

7. **IpcDataSource** (`packages/renderer/src/lib/data/IpcDataSource.ts`) — IPC call wrappers for both channels

8. **MockDataSource** (`packages/renderer/src/lib/data/MockDataSource.ts`) — minimal stub implementations

9. **DataProvider mutations** (`packages/renderer/src/lib/data/DataProvider.tsx`) — expose `mergePR()` and `openUrl()` via DataMutationValue

### UI wiring
10. **SessionDetail.tsx** — use `useDataMutations()` to wire:
    - Stop button → call `stopAgent(session.id)` (which already does optimistic status update)
    - "Open on GitHub" → call `openUrl(https://github.com/${pr.repo}/pull/${pr.number})`
    - Merge PR → call `mergePR(owner, repo, prNumber)` with error handling

11. **App.tsx** — wire Stop button in TaskView to call both `stopAgent()` and `updateSessionStatus()` for clean state transition (note: stopAgent already does optimistic update, so just calling stopAgent is sufficient)

## Notes
- `pr.repo` format is `"owner/repo"` (e.g. `"acme/ingest"`) — split to get owner and repo for Octokit
- GitHub URL construction: `https://github.com/${pr.repo}/pull/${pr.number}`
- StopAgent in DataProvider already does optimistic status update to "completed", so no separate call needed
