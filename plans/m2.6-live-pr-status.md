# M2.6: Live PR Status in Sidebar + Project View from Real Data

## Overview
Final milestone of Phase 2. Wire up the UI to display live PR data synced by M2.4 SyncService, replacing mock seed PRs with real `PullRequest[]` arrays.

## Changes

### 1. ProjectsSidebar.tsx
- Add `GitPullRequest` import from lucide-react
- Show PR count badge next to each project name (only when `p.prs.length > 0`)
- Badge format: `<icon>{openPRs} open` in a subtle secondary pill

### 2. OverviewTab.tsx
- Add "Pull Requests" section after References, before Activity
- Empty state: "No PRs synced yet — connect GitHub and run a sync to populate this view."
- Each PR row shows: mergeability dot (green/yellow/red), title + number, repo/branch, checks summary, status badge

### 3. AppShell.tsx
- Add `RefreshCw` import from lucide-react
- Import `useDataLoading` hook for sync state
- Sync button in project view header area: triggers `mutations.syncProject(projectId)`, shows spinner while syncing
- GitHub connection indicator near user avatar when not connected

### 4. DataProvider.tsx (useDataLoading hook)
- Extend return type to include `isSyncing` boolean from mutation context
- Currently `useDataLoading` only returns `{ loading, error }`; add `isSyncing` for AppShell sync button state
