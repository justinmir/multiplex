# M3.3: Session-scoped References persistence wiring

## Goal
Wire session-scoped references end-to-end: from the SessionDetail UI through DataProvider mutations to JsonRepository persistence via IPC.

## Changes

### 1. DataSource interface (`types.ts`)
- Add `upsertSessionReference(sessionId, reference)` and `deleteSessionReference(sessionId, refId)` convenience methods

### 2. IpcDataSource (`IpcDataSource.ts`)
- Implement new methods by passing `{ sessionId }` scope to existing IPC channels (`refs:upsert`, `refs:delete`)

### 3. DataProvider mutations (`DataProvider.tsx`)
- Add `upsertSessionReference` and `deleteSessionReference` to `DataMutationValue` interface
- Implement in provider — call data source methods, then trigger full refresh via loadData()

### 4. AppShell (`shell/AppShell.tsx`)
- Replace local-only `addReferenceToSession()` with real IPC-persisted mutation via `mutations.upsertSessionReference(sessionId, ref)`

### 5. MockDataSource (`MockDataSource.ts`)
- Add stub implementations for new interface methods (throw "not implemented")

## RefScope shape
Uses discriminator fields: `{ projectId: string; sessionId?: never }` or `{ sessionId: string; projectId?: never }`.
For session-scoped refs, pass `{ sessionId: "..." }`.
