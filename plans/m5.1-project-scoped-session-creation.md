# M5.1 — Project-Scoped Session Creation

**Phase:** 5 — Content Creation & Project Management  
**Depends on:** Phase 4 complete (M4.1–M4.4)

## Goal
Fix the broken project-scoped session creation flow. Currently clicking "New session" from within a project view renders a composer but does nothing when you click Start — no handler is wired to create and persist a project-linked session.

## Current State
- User clicks "New session" in ProjectView header → calls `openSession("new")` 
- This renders SessionsTab with `openId === "new"` 
- SessionsTab renders `<SessionDetail session={null} ... />` but does NOT pass `onStartSession`
- Composer's Send button delegates to `onStartSession?.(v)` which is undefined → nothing happens
- Standalone sessions work fine because AppShell passes `onStart={createSession}` via NewSessionView
- The IPC channel `sessions:create` accepts optional `projectId`, but UI never passes one

## Changes

### 1. SessionsTab.tsx: Pass through onStartSession callback
Add `onStartSession?: (prompt: string) => void` prop to SessionsTab and pass it down to SessionDetail's `onStartSession`.

### 2. ProjectView.tsx: Accept onCreateSession prop  
Add `onCreateSession?: (prompt: string, projectId: string) => void` prop from parent. Pass through to SessionsTab as `onStartSession={(prompt) => onCreateSession?.(prompt, project.id)}`.

### 3. AppShell.tsx: Implement project-scoped session creation handler
In the ProjectView rendering section (where we render project detail), pass a handler that:
- Creates a SessionMsg with role:"user" from the prompt text
- Builds a new Session object with projectId association 
- Calls `mutations.createSession(session, projectId)` to persist via IPC
- Navigates to the newly created session

The key difference from standalone sessions: this passes the optional `projectId` parameter so the session is associated with the project.

### 4. Verify persistence
When a project-scoped session is created, it should appear in both:
- The project's session list (via IPC filtering by projectId)  
- Potentially in the project activity log (if create handler already appends activity — check M3.1 implementation)
