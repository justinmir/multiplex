# M3.2: Live Data Subscriptions

## Goal
Wire the renderer's DataProvider to subscribe to `data:changed` IPC push events from the main process, so the UI automatically refreshes when data changes externally (e.g., another window or background process mutates data).

## Changes

### packages/renderer/src/lib/data/DataProvider.tsx
- Import `{ on }` from `"../ipc/client.js"` — the renderer-side IPC subscription wrapper
- Add a `useEffect` that subscribes to `"data:changed"` events, calling `loadData()` on each event
- The effect cleanup calls the unsubscribe function returned by `on()`
- Dependencies include `loadData` so we re-subscribe if the source changes

## How it works
1. Main process write handlers call `emit("data:changed", { kind: "note|session|..." })` (already implemented in M3.1)
2. Preload bridge receives events via `EVENT_CHANNEL` and dispatches to subscribers
3. IPC client's `on(topic, cb)` wraps preload's `subscribe()` — returns unsubscribe fn
4. DataProvider subscribes on mount; any `data:changed` event triggers a full data refresh
