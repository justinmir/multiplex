# M1.5: Write path — notes, references, session metadata persistence

## Goal
Add IPC write channels for creating/updating/deleting notes & references, and archiving sessions — all persisted to disk via JsonRepository. Wire mutations through DataProvider with optimistic updates. Wire handlers into existing component props in AppShell.

## Changes

### 1. `packages/core/src/ipc.ts`
- Import `Note`, `Reference` from domain (in addition to existing `Project`, `Session`)
- Add write channels to IpcContract:
  - `notes:upsert`, `notes:delete`
  - `refs:upsert`, `refs:delete`
  - `sessions:archive`

### 2. `packages/main/src/ipc/handlers/writes.ts` (new file)
- Create write handlers delegating to JsonRepository
- Each handler emits a `data:changed` event after the mutation
- Register via `registerRepoWriteHandlers(repo)`

### 3. `packages/main/src/modules/IpcModule.ts`
- Import and call `registerRepoWriteHandlers(repo)` alongside existing read handlers

### 4. `packages/renderer/src/lib/data/types.ts`
- Add write methods to DataSource interface:
  - `upsertNote`, `deleteNote`
  - `upsertReference`, `deleteReference`
  - `archiveSession`

### 5. `packages/renderer/src/lib/data/IpcDataSource.ts`
- Implement all write methods using IPC call()

### 6. `packages/renderer/src/lib/data/MockDataSource.ts`
- Implement noop write methods (keep existing file)

### 7. `packages/renderer/src/lib/data/DataProvider.tsx`
- Add DataMutationContext with mutation methods
- For project-scoped mutations: call refresh() after IPC write (ensures consistency without complex optimistic nested logic)
- For session archive: do optimistic local update + persist via IPC
- Export `useDataMutations()` hook

### 8. `packages/renderer/src/shell/AppShell.tsx`
- Wire `archiveSession` through IPC mutations (optimistic update + persist)
- Keep `addReferenceToSession` as local-only (session-scoped refs are Phase 3)
