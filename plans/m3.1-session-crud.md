# M3.1: Session CRUD via IPC

## Goal
Wire session creation and status updates through IPC to JsonRepository, replacing in-memory-only operations with persisted ones.

## Changes

### 1. `packages/core/src/ipc.ts` — Add IPC channels
- Import `SessionStatus` from domain.js
- Add `"sessions:create"` channel: req `{ session, projectId? }`, res `Session`
- Add `"sessions:update-status"` channel: req `{ sessionId, status }`, res `void`

### 2. `packages/main/src/ipc/handlers/session-writes.ts` — New handler file
- Register two IPC handlers following the pattern in writes.ts
- `sessions:create`: upsertSession + append activity + emit data:changed
- `sessions:update-status`: getSession, update status, upsertSession, optional activity for project-linked sessions, emit

### 3. `packages/main/src/modules/IpcModule.ts` — Wire handler registration
- Import and call `registerSessionWriteHandlers(this.repo)` in enable()

### 4. `packages/renderer/src/lib/data/types.ts` — Extend DataSource interface
- Add `createSession(session, projectId?): Promise<Session>`
- Add `updateSessionStatus(sessionId, status): Promise<void>`

### 5. `packages/renderer/src/lib/data/IpcDataSource.ts` — Implement IPC methods
- Wire through to IPC channels

### 6. `packages/renderer/src/lib/data/DataProvider.tsx` — Mutation context
- `createSession`: call data source + refresh via loadData()
- `updateSessionStatus`: optimistic update on standaloneSessions, rollback on error

### 7. `packages/renderer/src/shell/AppShell.tsx` — Wire createSession to IPC
- Replace `setSessions([next, ...prev])` with `mutations.createSession(next)`
